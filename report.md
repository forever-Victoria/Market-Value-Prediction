# 五大联赛足球运动员身价预测实验报告

**课程方向**：基于机器学习的体育主题数据分析  
**任务类型**：监督学习 · 回归（Regression）  
**数据来源**：Transfermarkt（[transfermarkt-datasets](https://github.com/dcaribou/transfermarkt-datasets)）

---

## 一、研究背景与问题定义

### 1.1 课题背景

现代职业足球转会市场资金规模巨大，球员身价的合理估计对俱乐部引援决策、薪资谈判与阵容规划具有现实意义。Transfermarkt 等机构基于竞技表现、合同期限、市场供需与专家评估给出参考身价，其背后存在可量化的规律，适合用数据驱动方法进行建模分析。

球员市场身价是衡量竞技价值与商业潜力的核心指标之一。转会决策、阵容构建均与身价估计密切相关。本课题选取欧洲五大联赛球员数据，在结构化表格上采用传统机器学习方法，完成从探索、预处理到建模评估的完整流程。

### 1.2 问题定义

本研究将球员身价预测形式化为**监督学习中的回归问题（Regression）**。

- 设特征向量为 $\mathbf{x}$（年龄、位置、联赛、国家队经历等），目标变量 $y$ 为连续数值 `market_value_in_eur`（欧元）。
- 任务是学习映射 $f: \mathbf{x} \mapsto \hat{y}$，使预测身价 $\hat{y}$ 尽可能接近真实身价 $y$。
- 该问题**不是分类任务**：输出不是离散类别（如高/中/低），而是连续金额。

### 1.3 研究目标

1. 对五大联赛（英超、西甲、德甲、意甲、法甲）球员数据进行清洗与特征工程；
2. 构建并对比三种回归模型，预测球员当前市场身价；
3. 使用 RMSE、MAE、$R^2$ 等指标评估模型，并分析关键影响因素；
4. 梳理线性回归、集成学习等相关理论，完成可复现的实验流程。

**研究范围**：数据限定为五大联赛；方法为 CPU 可运行的传统机器学习，不使用深度学习框架。

### 2.3 目标变量分布（长尾效应）

对身价绘制直方图（见 `outputs/01_eda_market_value.png` 左图）可见：绝大多数球员身价集中在较低区间，少数顶级球员身价极高，分布呈明显**右偏（长尾）**。

**专业解读**：均值易被极端高身价拉高；多数模型默认最小化平方误差，会被头部球星主导。因此后续对目标做 $\log(1+y)$ 变换，以稳定回归残差。

### 2.4 特征与身价的初步关系

年龄与身价散点图（同图右图，按联赛着色）显示：

1. **22–30 岁**区间高身价球员更集中，呈现**非线性**趋势，而非简单直线关系；
2. **五大联赛**在相同年龄段身价分布存在差异，联赛可作为重要特征；
3. 本版本已合并 `appearances.csv`，引入近期俱乐部进球、助攻与出场时间；无出场记录填 0。

### 2.5 特征增强（出场数据）

从 `appearances.csv` 聚合每位球员自 **2022-07-01** 起的：

- `recent_goals`, `recent_assists`, `recent_minutes`, `recent_apps`
- 派生：`goals_per90`（每 90 分钟进球）、`minutes_per_app`（场均分钟）

该步骤将逐场记录压缩为球员级竞技 proxy，是本次 **R² 从约 0.55 提升至 0.76** 的关键改进。

## 三、数据预处理与特征工程

### 3.1 数据清洗

1. **联赛筛选**：仅保留五大联赛球员；
2. **时效性**：`last_season ≥ 2023`，降低退役球员与过时身价噪声；
3. **目标过滤**：删除身价缺失或 ≤ 0 的记录；
4. **年龄派生**：由 `date_of_birth` 计算年龄，限制在 [16, 42]；
5. **标签泄漏防范**：剔除 `highest_market_value_in_eur`（历史最高身价与当前身价高度相关，使用会造成评估虚高）。

### 3.2 缺失值与异常值

| 类型 | 策略 | 理由 |
|------|------|------|
| 数值特征（身高等） | 中位数插补 | 对异常值比均值更稳健 |
| 分类特征 | 填 `Unknown` | 保留样本量 |
| 国家队出场/进球 | 缺失视为 0 | 未入选国家队即为 0 场 |
| 身价异常 | 删除 ≤ 0 或缺失 | 目标无效则无法监督学习 |

### 3.3 高基数分类变量

`country_of_citizenship` 出现次数 < 40 的国籍合并为 **Other**，降低独热编码维度、减轻过拟合。

### 3.4 特征编码（独热编码）

对 `position`、`foot`、`country_of_citizenship`、`current_club_domestic_competition_id` 使用 **One-Hot Encoding**：

- 将无序类别映射为 0/1 向量；
- 避免错误序关系（如把「西甲=3、英超=1」误认为 3 > 1）。

### 3.5 特征缩放（标准化）

对数值特征使用 **StandardScaler**：$z = (x - \mu) / \sigma$。

**原因**：

- 线性回归对特征尺度敏感；
- 年龄、身高、出场次数等量纲差异大，标准化后更利于优化与公平比较；
- 树模型虽不严格依赖缩放，统一 Pipeline 便于多模型公平对比。

**纳入特征**：

- 数值：`age`, `height_in_cm`, `international_caps`, `international_goals`, `last_season`, 以及 6 个近期出场统计字段
- 分类：`position`, `foot`, `country_of_citizenship`, `current_club_domestic_competition_id`

### 3.7 近期出场特征（关键改进）

从 `data/appearances.csv` 聚合自 **2022-07-01** 起的俱乐部比赛统计，合并至球员表：

| 字段 | 含义 |
|------|------|
| `recent_goals` / `recent_assists` | 进球、助攻合计 |
| `recent_minutes` / `recent_apps` | 出场时间、出场次数 |
| `goals_per90` | 每 90 分钟进球（效率指标） |
| `minutes_per_app` | 场均出场分钟 |

无出场记录的球员填 0。该步骤将 R² 从约 **0.55** 提升至 **0.76**。

### 3.8 训练/测试划分与 Pipeline

- 按 **8:2** 随机划分（`random_state=42`）。8:2 为常见默认：样本量充足时多留数据给模型学习，同时测试集（约 777 条）足以稳定估计指标；7:3 同样合理，本实验采用 8:2 遵循常见实践。
- 全部预处理步骤纳入 **Pipeline**，仅在训练集上 `fit`，测试集上 `transform`，防止**数据泄漏**。

### 3.9 目标变量变换

$$y' = \log(1 + y), \quad \hat{y} = \exp(\hat{y}') - 1$$

缓解身价右偏，压缩极端高身价对损失函数的主导。

## 四、模型理论与评估指标

### 4.1 回归任务形式化

给定特征 $\mathbf{x}$，学习 $f(\mathbf{x}) \approx y$。本实验为**监督学习**中的连续值预测。

### 4.2 算法原理

#### （1）线性回归（Linear Regression）—— 基线模型

假设 $\hat{y} = \mathbf{w}^\top \mathbf{x} + b$，通过最小化残差平方和（OLS）估计参数。

- **优点**：可解释性强、训练快；
- **局限**：难以刻画年龄、位置与身价之间的非线性与交互效应。

#### （2）随机森林（Random Forest）—— Bagging 集成

对训练集自助采样，训练多棵决策树，回归时取预测均值；每棵树在随机子特征上分裂。

- **优点**：捕捉非线性及特征交互、可输出特征重要性、抗过拟合；
- **超参**：`n_estimators=200`, `max_depth=16`, `min_samples_leaf=5`。

#### （3）梯度提升树（Gradient Boosting）—— Boosting 集成

串行训练浅层决策树，每棵拟合前一轮残差，逐步降低损失；通过 learning rate 与 subsample 控制过拟合。

- **优点**：结构化表格回归中常优于 Bagging 与线性模型；本实验表现最佳；
- **超参**：`n_estimators=400`, `max_depth=6`, `learning_rate=0.05`, `subsample=0.8`, `min_samples_leaf=3`。

### 4.3 回归评估指标

| 指标 | 公式 | 含义 |
|------|------|------|
| **MAE** | $\frac{1}{n}\sum_{i=1}^{n}\|y_i - \hat{y}_i\|$ | 平均绝对误差，典型偏差多少欧元，**越小越好** |
| **RMSE** | $\sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}$ | 对大误差更敏感，回归最常用指标之一，**越小越好** |
| **$R^2$** | $1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}$ | 相对「始终猜平均身价」的提升，**越接近 1 越好**；0 表示与猜均值相当 |

### 4.4 实验设置

- 实现文件：`player_market_value_ml.py`
- 环境：CPU + scikit-learn，不使用 PyTorch / TensorFlow

## 五、实验结果与对比分析

### 5.1 模型性能对比

**测试集结果**（来源：`outputs/model_metrics.csv`）：

| 模型 | RMSE（欧元） | MAE（欧元） | $R^2$ |
|------|-------------|------------|-------|
| 线性回归 | 9,912,534 | 4,273,227 | 0.675 |
| 随机森林 | 9,832,515 | 3,903,541 | 0.681 |
| **梯度提升树** | **8,460,391** | **3,613,959** | **0.764** |

**可视化**：`outputs/02_model_metrics_comparison.png`

### 5.2 结果分析

1. **线性回归 $R^2 \approx 0.68$**：加入出场数据后，线性模型亦显著提升，说明竞技统计与身价存在较强线性成分；但仍弱于树模型。
2. **随机森林 $R^2 \approx 0.68$**：与线性接近，非线性交互仍有增益空间。
3. **梯度提升树最优**：$R^2 \approx 0.76$，RMSE 约 **846 万欧元**，较仅使用 players 表特征（$R^2 \approx 0.55$）提升约 **0.21**。
4. **误差量级**：MAE 约 361 万欧元；引入出场数据后典型偏差明显收窄。

### 5.3 特征重要性

**可视化**：`outputs/03_feature_importance.png`（梯度提升树 Top 10）

| 特征 | 业务解读 |
|------|----------|
| `recent_apps` / `recent_minutes` | 近期出场频次与时间：竞技参与度直接关联估值 |
| `age` | 竞技生命周期：黄金年龄段溢价更高 |
| `recent_goals` / `goals_per90` | 进球产出是攻击型球员溢价的核心 |
| `international_caps` | 国家队资历是实力与商业价值的信号 |
| `position_*` / 联赛变量 | 位置稀缺性与联赛平台溢价 |

### 5.4 业务洞察

实验表明，**近期俱乐部出场与进球产出**是身价预测最强信号，年龄与国家队经历次之，联赛与位置提供平台与结构性溢价。引入 `appearances.csv` 后，模型从「只能解释约 55% 变异」提升到「约 76%」，说明竞技表现数据对 Transfermarkt 估值具有决定性解释力。

### 5.5 特征消融实验（Ablation Study）

为验证「特征重要性」不仅是统计相关，且对预测性能具有实质贡献，在最佳模型（梯度提升树）上增加消融实验：保持同一随机划分与超参数，分别用完整特征、剔除核心特征、分组特征重训并对比测试集指标。

**理论说明**：消融实验（Ablation Study）通过 controlled 对比回答——*去掉某类信息后，模型变差多少？* 若某特征在重要性图中排名靠前，且剔除后 $R^2$ 显著下降，则形成「重要性排序 + 性能验证」的闭环论证。

**方案 A（剔除 Top-3 重要原始特征）**  
根据完整模型的特征重要性，聚合后 Top-3 为：`recent_apps`、`recent_minutes`、`age`。

**方案 B（业务分组）**  
- 仅基础信息：年龄、身高、位置、惯用脚  
- 仅生涯/竞技 proxy：国家队、最近赛季、近期俱乐部出场统计  
- 仅近期俱乐部表现：appearances 聚合字段  
- 仅市场/背景：国籍、所属联赛  

**消融结果**（`outputs/ablation_metrics.csv`，图见 `outputs/04_ablation_study.png`）：

| 消融场景 | $R^2$ | RMSE（欧元） | $R^2$ 相对基准 | RMSE 升幅 |
|----------|-------|-------------|----------------|-----------|
| 完整特征（基准） | 0.764 | 8,460,391 | — | — |
| 剔除 Top-3（recent_apps, recent_minutes, age） | 0.506 | 12,225,539 | **−0.257** | **+44.5%** |
| 仅基础信息 | −0.025 | 17,609,289 | −0.788 | +108.1% |
| 仅生涯/竞技 proxy | 0.527 | 11,970,814 | −0.237 | +41.5% |
| 仅近期俱乐部表现 | 0.400 | 13,471,321 | −0.363 | +59.2% |
| 仅市场/背景 | −0.027 | 17,634,504 | −0.791 | +108.4% |

**分析结论**：

1. **方案 A**：剔除 Top-3 后 $R^2$ 从 0.76 跌至 0.51，RMSE 恶化 **44.5%**，证明出场与年龄是核心输入。  
2. **方案 B**：「仅近期俱乐部表现」仍可达 $R^2 \approx 0.40$，高于「仅市场背景」；与完整模型差距说明需与年龄、位置等联合使用。  
3. **完整特征必要性**：各类信息互补，联合使用才能达到 $R^2 \approx 0.76$。

**答辩可用表述**：  
> 引入出场数据后，recent_apps、recent_minutes 与 age 成为最重要特征；消融实验证明剔除 Top-3 后 RMSE 上升约 45%，验证了竞技参与度对身价预测的决定性作用。

## 六、结论与局限性

### 6.1 总结

本研究基于 Transfermarkt 五大联赛 **3,884** 名球员数据，完成 EDA → 预处理（含出场特征聚合）→ 三模型对比 → 消融实验的完整回归流水线。测试集上**梯度提升树**表现最优（$R^2 \approx 0.76$，RMSE 约 **846 万欧元**），较仅使用 players 表基础特征（$R^2 \approx 0.55$）显著提升，验证了「竞技表现 + 人口统计 + 联赛平台」联合建模的有效性。

### 6.2 局限性

| 局限 | 说明 |
|------|------|
| 未纳入合同、伤病、舆情等 | 仍有约 24% 变异无法解释 |
| 出场统计为跨赛季聚合 | 可改为「最近一个完整赛季」提高时效一致性 |
| **标签噪声** | Transfermarkt 身价为估计值，非实际成交价 |
| **划分与超参** | 固定 8:2 划分，未做交叉验证与系统调参 |
| **极端身价** | 少数超级球星拉高 RMSE；可考虑分位数回归或分段建模 |

### 6.3 改进方向

- 将出场窗口收紧为「最近一个完整赛季」，并与 `last_season` 对齐；
- 引入合同剩余年限、俱乐部总身价、伤病记录等；
- 使用 `GridSearchCV` 做系统化超参搜索；
- 按位置（前锋/后卫/门将）分组建模，改善两端预测精度。

## 附录 C：答辩常见问题

**Q1：为什么不用深度学习？**  
数据为结构化表格、样本约四千级，传统树模型已是强基线且可解释；课程要求 CPU 环境，无需神经网络。

**Q2：$R^2$ 约 0.76，算好吗？**  
引入出场数据后已能解释约 76% 身价变异；未观测因素（合同、伤病、舆情）限制进一步上限。重点在**相对对比**与**特征工程 + 消融验证**。

**Q3：为什么剔除历史最高身价？**  
与当前身价高度相关，使用会造成**标签泄漏**，测试集指标会虚高。

**Q4：为什么用 8:2 而不是 7:3？**  
均为常见划分；8:2 在样本充足时多留数据训练，测试集约 777 条已够稳定评估。7:3 测试更稳、训练略少，本实验取 8:2 为常见默认。

**Q5：什么是消融实验？为什么要做？**  
通过去掉某组特征后观察 $R^2$/RMSE 变化，验证特征重要性的结论。剔除 Top-3（recent_apps、recent_minutes、age）后 RMSE 上升约 **45%**。
